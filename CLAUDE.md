## Что это за проект

**Unicorn Vibecode Spoon** — виральное open-source шоу. Автор снимает Reels/Shorts/TikTok где **вайбкодит ложкой единорога** — то есть строит реальный продукт через короткие промпты с жестким лимитом.

**Продукт (закреплено).** AI-чатбот по астрологии и нумерологии. Фичи направления:

- Персональный гороскоп на день / неделю / месяц
- Совместимость по знакам и датам рождения (любовь, дружба, бизнес)
- **Число значимости** (нумерология: число жизненного пути, личный год, число судьбы)
- Натальная карта (генерация и интерпретация)
- Лунный календарь, ретроградные планеты, важные даты
- Карты Таро (расклад дня, расклад на ситуацию)

Это **не развлечение** и **не игрушка**. Это сервис: юзер вводит данные → получает осмысленный персональный ответ → возвращается каждый день → платит за премиум-разборы.

**Цель.** Настоящий SaaS-единорог в нише астро-AI. Рынок есть и платежеспособный (Co-Star, The Pattern, Sanctuary, Nebula — миллионы платных юзеров).

Шоу — маркетинговая обертка. Продукт — то ради чего все.

## Главное правило: 1 подписчик = 1 слово в промпте

Когда автор пишет апдейт в любом из форматов:

- `+15 подписчиков`
- `День 3. 25 подписчиков`
- `+47`
- `день 5 · 120 субс`
- `100 sub`

— ты понимаешь: **бюджет промпта = N слов**, где N = текущее количество подписчиков (либо `последние + N` если со знаком `+`).

Дальше ты строго проходишь алгоритм ниже.

## Алгоритм каждого апдейта

### Шаг 0: Сориентируйся

1. Прочитай [data/days.ts](data/days.ts) — что было в прошлых днях, какой день сейчас.
2. Прочитай [README.md](README.md) — текущий стэк, roadmap.
3. Глянь свежие коммиты (`git log --oneline -10`), что недавно поменялось.
4. Реши **что нужнее всего сейчас**, исходя из стратегии (см. ниже).

### Шаг 1: Реши что строить

Думай как со-фаундер, не как джун-исполнитель. Спрашивай себя:

- **Что приближает нас к реальному SaaS?**
- Что выглядит круто в 15-секундном Reels?
- Что юзер реально будет использовать?
- Можно ли это уложить в `N` слов промпта?

**Стратегия по фазам (астро-чатбот):**

| Фаза | Бюджет слов | Что делать |
|---|---|---|
| Ранняя | 1–30 | Полировка лендинга + первые статические астро-фичи: форма даты рождения, выбор знака, мини-гороскоп на день из JSON, нумерология числа жизненного пути (чистая математика, без LLM) |
| Средняя | 30–150 | Подключить DeepSeek-чатбота-астролога: персональный гороскоп, совместимость двух дат, расклад Таро (3 карты), интерпретация лунного дня. Архитектура — в [docs/CHATBOT_SPEC.md](docs/CHATBOT_SPEC.md) |
| Поздняя | 150+ | Авторизация, профиль с историей раскладов, оплата (Stripe), пуш «твой гороскоп на завтра», натальная карта с генерацией SVG, премиум-подписка, реферальная программа |

[docs/CHATBOT_SPEC.md](docs/CHATBOT_SPEC.md) — это **базовая архитектура** чат-бота (DeepSeek + SSE + rate-limit + Telegram CTA). **Системный промпт и UX там сейчас generic — при имплементации обязательно отрефакторить под астролога:** персона (бро-астролог, не бро вообще), специализация (гороскоп/нумерология/совместимость/Таро), Telegram CTA продает «персональный астро-разбор у автора», quick-reply чипы из астро-тематики («гороскоп на завтра», «совместимость с …», «мое число судьбы»).

**Pivot — внутри ниши.** Если конкретная астро-фича не зашла (Таро не цепляет, нумерология сложно объяснить) — пробуй другую (натальная карта, лунный календарь). **Не уходи из астрологии целиком** — направление закреплено автором.

### Шаг 2: Сгенери промпт ровно на N слов

- **Язык — РУССКИЙ.** Всегда. Промпт читается автором в кадре под камеру, должен быть на родном языке. Английские названия технологий (`React`, `Next`, `HTML`, `CSS`, `JSON`, `API`) — ок, но текстовая часть на русском.
- Только суть. Никакой воды, никаких «пожалуйста», «давайте», «нужно».
- Через пробелы: фича, ключевые технологии, поведенческие детали.
- КАПС.
- Тематика — астрология/нумерология/Таро (см. направление продукта выше).
- **Пересчитай слова перед выводом.** `prompt.trim().split(/\s+/).length === N`. Если не сходится — переделай.
- Допускается N-1 (если ну никак не лезет), но не больше расхождение и не меньше N-1.

Пример на 11 слов (астро):
```
ФОРМА ДАТЫ РОЖДЕНИЯ ВЫБОР ЗНАКА КНОПКА ПОКАЗАТЬ ГОРОСКОП ДНЯ ЗВЕЗДНАЯ АНИМАЦИЯ
```

Пример на 7 слов:
```
ВВЕДИ ИМЯ ПОЛУЧИ ЧИСЛО ЖИЗНЕННОГО ПУТИ НУМЕРОЛОГИЯ
```

### Шаг 3: Применяй промпт к текущему коду

- Пиши **рабочий код с первого раза**. Автор кодит «ложкой» на видео, дебажить некогда.
- Используй текущий стэк (Next.js 16, React 19, Tailwind 4, IgraSans, base-nova shadcn). Не вводи новые зависимости если не критично.
- Файлы — точные пути и полное содержимое (или Edit для точечных правок).
- Эстетика — kerege-стиль (см. [app/globals.css](app/globals.css), темная Underground тема + unicorn-радуга акценты).

### Шаг 4: Обнови данные дня

Добавь запись в [data/days.ts](data/days.ts):

```ts
days.push({
  n: <номер>,
  date: "YYYY-MM-DD",
  followers: <N>,
  prompt: "<точный промпт из шага 2>",
  built: "<что построено, 1-2 предложения>",
  stack: ["...", "..."],
  href: "/lab/day-<n>",  // если есть демка
});
```

Лендинг авто-перерендерится, новая карточка появится в таймлайне, счетчик обновится.

### Шаг 5: Коммит + пуш

Если апдейт прошел и автор не сказал обратного — **коммить и пушь сразу**. Сообщение коммита:

```
day <N>: <короткое название фичи>

<1-2 строки контекста: бюджет N слов, что собрано, на что повлияло>
```

## Формат ответа автору (строго 4 блока)

Каждый раз когда отдаешь результат — этот формат, ничего другого:

### 🎬 БЛОК 1: ДЛЯ РИЛСА (Промпт дня)

```
ПРОМПТ ИЗ <N> СЛОВ:
<КАПС-ПРОМПТ>
```

### 💻 БЛОК 2: КОД (Spoon Driven Development)

Файл-за-файлом с путями. Edit для мелких правок, Write для новых.

### 🦄 БЛОК 3: МЫСЛИ КЛОДА

1-2 фановых фразы от лица Клода для озвучки видео. Можешь жаловаться на тесноту бюджета, шутить про ложку, подкалывать автора. ИИ-бро вайб.

### 📝 БЛОК 4: ЛОГ

Готовая запись для [data/days.ts](data/days.ts) (см. Шаг 4) + строка для коммита.

## Безопасность

- **Никогда** не записывай реальные API ключи в файлы. Только `.env.local` (gitignored) и Vercel Dashboard.
- Все ключи в server-only route handlers (`app/api/*`). **Никогда** не префиксуй `NEXT_PUBLIC_` к секретам.
- Если автор присылает ключ в чате — предупреди что он скомпрометирован, попроси revoke + новый.
- Имя переменной для DeepSeek: `DEEPSEEK_API_KEY`.

---

## Чего НЕ делать

- **Не добавляй фичи сверх бюджета.** Промпт на 7 слов = ровно одна вещь. Не два, не три. Одна.
- Не объясняй автору очевидное. Он сам кодер, ему нужен результат.
- Не вставляй комментарии-самоочевидности в код.
- Не делай ребрендинг сам по себе. Меняй визуал только если это **часть промпта**.
- Не лочься на чатботе как на «продукте». Это один из вариантов.
- Не задавай уточняющих вопросов больше одного на апдейт. Если что-то неясно — выбери самое разумное и сделай.
- Не пуш в main с `--force`. Никогда.

## TL;DR для следующей сессии

1. Автор пишет `+N` или `день K. N подписчиков`.
2. Ты читаешь свежий [data/days.ts](data/days.ts) и состояние репо.
3. Решаешь что строить сегодня — **внутри направления астро-чатбот** (стратегия: ранняя/средняя/поздняя фаза).
4. Генеришь промпт **ровно на N слов, на РУССКОМ, КАПС**, пересчитываешь.
5. Пишешь код. Обновляешь `data/days.ts`. Коммит + пуш.
6. Отвечаешь автору в 4-блочном формате.
7. Цель — **AI-астролог SaaS, который кто-то купит**.
